## Bubblewrap Signing Behavior

Bubblewrap signs through `bubblewrap build` when `--skipSigning` is omitted.

Supported signing overrides:

```sh
bubblewrap build --signingKeyPath "<path-to-keystore>" --signingKeyAlias "<alias>"
```

Bubblewrap reads the keystore path and alias from `android/twa-manifest.json` by default, but `--signingKeyPath` and `--signingKeyAlias` override those values. Prefer the overrides so the repo does not need to store a machine-specific signing path.

Bubblewrap asks for two passwords unless both environment variables are set:

- `BUBBLEWRAP_KEYSTORE_PASSWORD`
- `BUBBLEWRAP_KEY_PASSWORD`

Do not commit passwords or write them to tracked files.

Signed outputs generated by Bubblewrap:

- Signed AAB: `android/app-release-bundle.aab`
- Signed APK: `android/app-release-signed.apk`

These outputs are ignored and must not be committed.

## Commands To Run After Approval

Create the signing directory:

```powershell
New-Item -ItemType Directory -Force -Path "$env:USERPROFILE\Documents\bubby-signing"
```

Create the upload key:

```powershell
keytool -genkeypair -v `
  -keystore "$env:USERPROFILE\Documents\bubby-signing\bubby-upload-key.jks" `
  -alias "bubby-upload" `
  -keyalg RSA `
  -keysize 2048 `
  -validity 20000 `
  -dname "CN=Tim Yenin, OU=Bubby, O=Bubby, L=Atlanta, ST=Georgia, C=US"
```

Then build signed artifacts from the Android wrapper directory:

```powershell
cd android
bubblewrap build `
  --signingKeyPath "$env:USERPROFILE\Documents\bubby-signing\bubby-upload-key.jks" `
  --signingKeyAlias "bubby-upload"
```

If using environment variables instead of interactive password prompts, set them only in the current shell and clear them afterward:

```powershell
$env:BUBBLEWRAP_KEYSTORE_PASSWORD = "<keystore-password>"
$env:BUBBLEWRAP_KEY_PASSWORD = "<key-password>"
cd android
bubblewrap build --signingKeyPath "$env:USERPROFILE\Documents\bubby-signing\bubby-upload-key.jks" --signingKeyAlias "bubby-upload"
Remove-Item Env:\BUBBLEWRAP_KEYSTORE_PASSWORD
Remove-Item Env:\BUBBLEWRAP_KEY_PASSWORD
```

Do not paste real passwords into tracked docs, scripts, or shell history if avoidable.

## Fingerprints And Digital Asset Links

There are two important signing identities:

- Local upload key: the keystore created above. Its SHA-256 fingerprint may be useful for local sideload testing.
- Google Play App Signing certificate: the certificate Google Play uses to sign the app delivered to users.

For the production Trusted Web Activity relationship, use the Play App Signing certificate SHA-256 fingerprint from Play Console once it is available. Do not create `public/.well-known/assetlinks.json` with a placeholder or guessed fingerprint.

Useful fingerprint command after the keystore exists:

```powershell
keytool -list -v `
  -keystore "$env:USERPROFILE\Documents\bubby-signing\bubby-upload-key.jks" `
  -alias "bubby-upload"
```

Keep the keystore and passwords private and backed up outside the repo.
